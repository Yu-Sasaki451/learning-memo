Fortify

全体像

Fortifyはログイン、登録、ログアウトなどの認証処理を用意してくれる。
画面は自分でBladeを作り、FortifyのルートへPOSTする。

手順1 Fortifyを入れる

Usersテーブルがある前提で、Fortifyをインストールしてマイグレーションする。

composer require laravel/fortify
php artisan fortify:install
php artisan migrate

手順2 Bladeを紐づける

app/Providers/FortifyServiceProvider.php の boot() に追加する。

use Laravel\Fortify\Fortify;

public function boot(): void
{
    Fortify::loginView(function () {
        return view('auth.login');
    });

    Fortify::registerView(function () {
        return view('auth.register');
    });
}

手順3 フォームを作る

Fortifyが用意しているログイン処理はPOST /login。
登録処理はPOST /register。

ログインフォーム

<form method="post" action="/login">
    @csrf
    <input type="email" name="email">
    <input type="password" name="password">
    <button type="submit">ログイン</button>
</form>

登録フォーム

<form method="post" action="/register">
    @csrf
    <input type="text" name="name">
    <input type="email" name="email">
    <input type="password" name="password">
    <input type="password" name="password_confirmation">
    <button type="submit">登録</button>
</form>

手順4 登録処理を確認する

登録処理はapp/Actions/Fortify/CreateNewUser.phpで調整する。
passwordにconfirmedを使う場合は、フォームにpassword_confirmationが必要。 

public function create(array $input): User
{
    Validator::make($input, [
        'name' => ['required', 'string', 'max:255'],
        'email' => ['required', 'string', 'email', 'max:255', 'unique:users'],
        'password' => ['required', 'string', 'confirmed', Rules\Password::defaults()],
    ])->validate();

    return User::create([
        'name' => $input['name'],
        'email' => $input['email'],
        'password' => Hash::make($input['password']),
    ]);
}

手順5 ログイン後の遷移先

ログイン後の遷移先はconfig/fortify.phpのhomeで設定する。

config/fortify.php

'home' => '/admin',

routes/web.php

Route::middleware('auth')->group(function () {
    Route::get('/admin', [UserController::class, 'showAdmin']);
});
