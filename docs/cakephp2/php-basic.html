PHP基礎とCakePHP2でよく見る書き方

会社コードで出てきやすいPHPの記号、配列、条件分岐、文字列連結を読むためのページです。
まず生のPHPを理解してから、CakePHP2のController、Model、Viewでどう出てくるかを見ます。

まず読む順番

PHPとCakePHP2のコードは、記号を1つずつ見るより「値を作る」「条件で分ける」「画面やDBへ渡す」の順で読むと分かりやすいです。

1. 変数 $name のように、値に名前を付けている場所を確認します。
2. 配列 ['User']['name'] のように、どの箱の中を見ているか確認します。
3. 条件 if、empty()、=== で処理が分かれる条件を読みます。
4. 受け渡し return、$this->set()、save() で値がどこへ行くか見ます。

PHPファイルの基本形

この処理は、PHPを書く場所を示す基本形です。
<?php からPHPが始まり、命令の最後には ; を付けます。 

<?php
$name = '田中';
$age = 30;

echo $name;
echo $age;
<?php ここからPHPを書くという開始記号。Viewの .ctp でもよく出ます。
$name 変数。値に名前を付けて、あとから使えるようにします。
= 右側の値を左側の変数へ入れる記号です。比較ではなく代入です。
; PHPの命令の終わりを示します。付け忘れると構文エラーになります。
echo 値を画面へ出力します。ViewでHTMLに値を表示する時によく使います。

文字列と数値

この処理は、文字と数値の違いを示しています。
PHPでは、文字列はクォートで囲み、数値はそのまま書きます。 

<?php
$name = '田中';
$age = 30;
$message = 'ログインしました';
'田中' 文字列です。名前、メッセージ、日付文字列などはこの形で書きます。
30 数値です。計算や大小比較に使えます。
'30' 数字に見えても、クォートで囲むと文字列です。=== では数値の30とは別物です。

配列 array() と []

この処理は、複数の値を1つの変数にまとめる書き方です。
CakePHP2の古いコードでは array() が多く、最近のPHPでは [] もよく使われます。 

<?php
$data = array(
    'name' => '田中',
    'age' => 30
);

$sameData = [
    'name' => '田中',
    'age' => 30
];
array(...) 配列を作る古い書き方です。CakePHP2のサンプルや会社コードでよく見ます。
[...] 配列を作る短い書き方です。意味は array() と同じです。
'name' => '田中' name というキーに、田中 という値を入れています。
=> 配列の「キー」と「値」をつなぐ記号です。左が名前、右が中身です。

連想配列

この処理は、文字列のキーで値を保存する配列です。
エラー結果、ログイン結果、Modelの検索条件などでよく使います。 

<?php
$result = [
    'status' => 'error',
    'msg' => null,
    'posted_data' => null
];

$status = $result['status'];
$result 処理結果をまとめて入れている変数です。
'status' 配列のキーです。どの値を取り出すかを指定する名前です。
null 値がまだない、何も入っていない、という意味です。
$result['status'] $result 配列の中にある status の値を取り出します。

多次元配列

この処理は、配列の中にさらに配列が入っている状態です。
CakePHP2のModel取得結果では、$userData['Nursery']['id'] のような形がよく出ます。 

<?php
$userData = [
    'Nursery' => [
        'id' => 10,
        'force' => 0
    ]
];

$nurseryId = $userData['Nursery']['id'];
$userData['Nursery'] $userData の中にある Nursery 配列を見ています。
['id'] Nursery 配列の中にある id を見ています。
$nurseryId 取り出したIDを入れておく変数です。この例では10が入ります。

配列へ値を追加する

この処理は、保存用の配列にあとから値を追加しています。
CakePHP2では、Modelの save() へ渡す前に保存データを作る時によく出ます。 

<?php
$saveData = [
    'logged_in' => date('Y-m-d H:i:s')
];

$saveData['force'] = 0;
$saveData['force'] = 0; force というキーを追加し、値として0を入れています。
date('Y-m-d H:i:s') 現在日時を 2026-07-16 15:30:00 のような形で作ります。
注意点

+ で配列を足す書き方もありますが、同じキーがある場合は左側の値が優先されます。上書きしたい時は $saveData['force'] = 0; の方が読みやすいです。

<?php
$saveData += [
    'force' => 0
];

文字列の連結 . と .=

この処理は、文字列をつなげています。
PHPでは文字列をつなぐ時に . を使います。 

<?php
$name = '田中';
$text = '名前：' . $name;

$output = 'A';
$output .= 'B';
. 左の文字列と右の文字列をつなげます。この例では 名前：田中 になります。
.= 今ある文字列の後ろに、右側の文字列を追加します。この例では AB になります。
$output .= 'B'; $output = $output . 'B'; と同じ意味です。

HTMLをPHPの文字列として作る

この処理は、PHPの文字列としてHTMLタグを作っています。
CakePHP2の古いViewやHelper周辺では、文字列を連結してリンクや表示文を作るコードがあります。 

<?php
$tmp = '<a id="approval_10_1_2026-07-16" href="#">承認</a>';

echo $tmp;
$tmp HTMLの <a> タグを文字列として入れている変数です。
echo $tmp; 文字列を画面に出します。この場合、画面では「承認」というリンクとして表示されます。
id="approval_10_1_2026-07-16" JavaScriptやCSSからこのリンクを指定するための名前です。

条件分岐 if

この処理は、条件を満たした時だけ中の処理を実行します。
if の丸括弧の中が成立した場合だけ、波括弧の中へ進みます。 

<?php
if ($userData['Nursery']['force'] < 0) {
    $saveData['force'] = 0;
}
if (...) 条件を確認する書き方です。条件が正しい時だけ中の処理が動きます。
< 0 0より小さいかを確認しています。
$saveData['force'] = 0; force がマイナスなら、保存する値を0に直しています。

&& は両方成立

この処理は、2つの条件が両方とも成立しているか確認しています。
&& は「かつ」という意味です。 

<?php
if (
    !empty($tmpLogin[1])
    && $tmpLogin[1] === APP_ACCOUNT_MAINTENANCE
) {
    $mode = 'maintenance';
}
!empty($tmpLogin[1]) $tmpLogin[1] が空ではないかを確認しています。! は否定です。
&& 左の条件と右の条件が両方成立した時だけ、全体が成立します。
=== 値だけでなく型も含めて完全に一致するか確認します。
APP_ACCOUNT_MAINTENANCE 定数です。固定の名前に固定の値を持たせているものです。

|| はどちらか成立

この処理は、2つの条件のどちらか1つでも成立しているか確認しています。
|| は「または」という意味です。 

<?php
if ($role === 'admin' || $role === 'manager') {
    $canApprove = true;
}
$role === 'admin' 権限がadminか確認しています。
|| 左か右のどちらかが成立すれば、全体が成立します。
$role === 'manager' 権限がmanagerか確認しています。
$canApprove = true; 承認できる状態として、真偽値のtrueを入れています。

empty() と !empty()

この処理は、変数が空かどうかを確認しています。
empty() は空ならtrue、!empty() は空ではないならtrueです。 

<?php
if (!empty($result)) {
    $status = $result['status'];
}
empty($result) $result が空かどうかを確認します。
!empty($result) $result が空ではないかを確認します。
空として扱われやすい値 null、空文字、0、空配列、false などです。

関数とreturn

この処理は、まとまった処理に名前を付けて、結果を呼び出し元へ返しています。
return は「この関数の結果はこれです」と返す命令です。 

<?php
function getName()
{
    return '田中';
}

$name = getName();
function getName() getName という関数を作っています。
return '田中'; 呼び出し元へ 田中 という文字列を返しています。
$name = getName(); 関数の戻り値を $name に入れています。

list() で複数の戻り値を受け取る

この処理は、配列で返された複数の値を、それぞれ別の変数へ入れています。
古いPHPやCakePHP2の会社コードでは list() が出ることがあります。 

<?php
list($result, $userData, $authValid)
    = $this->authCheck($loginId, $passWord, $office);

この処理は、次のように配列で返された値を順番に受け取る前提です。 

<?php
return [
    $result,
    $userData,
    $authValid
];
$result 戻り値の1番目が入ります。
$userData 戻り値の2番目が入ります。
$authValid 戻り値の3番目が入ります。
注意点 順番で代入されるため、返す配列の順番と list() 側の順番を合わせる必要があります。

key()

この処理は、配列の現在位置にあるキーを取得しています。
値ではなく、配列のキー名を取りたい時に使われます。 

<?php
$serviceData = [
    '保育サービス' => [
        'service' => 1
    ]
];

$serviceName = key($serviceData);
$serviceData サービス名をキーにした配列です。
key($serviceData) 現在位置のキーを取得します。この例では 保育サービス が入ります。
注意点 配列の中身ではなくキーを取っています。service の値である1を取っているわけではありません。

mb_substr()

この処理は、日本語を含む文字列の一部を切り出しています。
mb_ で始まる関数は、日本語のようなマルチバイト文字を扱う時によく使います。 

<?php
$text = 'あいうえお';
$result = mb_substr($text, 0, 3);
$text 切り出し元の文字列です。
0 開始位置です。0は先頭を意味します。
3 取り出す文字数です。この例では あいう になります。

HTMLのidとJavaScriptの#

この処理は、HTMLの特定の要素をJavaScriptで指定しています。
HTML側の id をJavaScriptやCSSで指定する時は、先頭に # を付けます。 

<a id="approval_10_1_2026-07-16">承認</a>
$('#approval_10_1_2026-07-16')
id="approval_10_1_2026-07-16" HTML要素に付けた名前です。同じページ内では基本的に重複させません。
#approval_10_1_2026-07-16 JavaScriptやCSSで、そのidを持つ要素を指定する書き方です。

セレクタ文字列を作る

この処理は、複数のHTML要素をまとめて選択するための文字列を作っています。
これは配列ではなく、カンマ区切りの1つの文字列です。 

<?php
$approvalList .=
    '#approval_' .
    $nurseryId . '_' .
    $thisData['service'] . '_' .
    $thisData['date'] .
    ',';
$approvalList .= 既存の文字列の後ろに、新しいセレクタ文字列を追加しています。
'#approval_' JavaScriptやCSSでidを指定するための先頭部分です。
. 文字列や変数をつなげています。
',' 複数のセレクタを区切るためのカンマです。
作られる文字列の例

#approval_10_2_2026-07-16,#approval_11_3_2026-07-17,

最後に余ったカンマは、mb_substr() などで削除することがあります。

文字列とHTMLを組み合わせる

この処理は、サービス名、括弧、リンクHTMLをつなげて表示用の文字列を作っています。
画面ではリンク部分だけクリックできる文字として表示されます。 

<?php
$output .= '' . key($serviceData) . '（' . $tmp . '）';
$output .= 表示用の文字列に、今回作った文字列を追加しています。
key($serviceData) 配列のキーからサービス名を取り出しています。
'（' . $tmp . '）' 全角括弧の中に、$tmp のリンクHTMLを入れています。
完成例

延長保育（<a href="#">承認</a>）

画面では 延長保育（承認） のように見え、承認の部分がリンクになります。

CakePHP2のController / Model / View

この処理は、CakePHP2で役割が分かれる3つの場所です。
PHPの基礎を読めるようになると、どのファイルで何をしているか追いやすくなります。

Controller 画面からのリクエストを受け取り、Modelを呼び、Viewへ値を渡す場所です。例は app/Controller/UsersController.php です。
Model データベースの検索、登録、更新、削除を担当します。例は $this->Nursery->save($saveData) です。
View ユーザーに表示するHTMLを作る場所です。CakePHP2では .ctp に生のPHPを書くことがよくあります。

公式で確認したこと

PHP公式マニュアルでは、配列はキーと値を結び付ける構造で、array() と短い [] の両方が説明されています。
文字列連結は .、追記代入は .= です。CakePHP2公式では、Viewファイルは通常 .ctp で、プレーンなPHPで書かれると説明されています。
