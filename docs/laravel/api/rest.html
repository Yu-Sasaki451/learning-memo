Laravel API作成

LaravelでJSONを返すAPIを作る基本。
Reactから呼び出せるTodo APIを例にする。

公式で確認したこと

Laravel 12系の公式では、API用ルーティングはinstall:apiで用意できる。
APIの返却形式を整えるときは、Eloquent API Resourcesを使う。

  • Laravel公式 Routing
  • Laravel公式 Controllers
  • Laravel公式 Requests
  • Laravel公式 Eloquent API Resources
  • Laravel公式 Validation
  • Laravel公式 Sanctum
$todo $ は変数 PHPでは変数の名前の前に$を書く。
Todo::create() :: はクラスから呼ぶ Todoモデルのcreateを呼んでいる。
$todo->update() -> は中の処理を呼ぶ 作成済みの$todoに対してupdateを実行する。
['title' => ['required']] => は対応 titleにrequiredルールを対応させている。

APIルートを使えるようにする

Laravel 11以降の構成では、APIを使う前にinstall:apiを実行する。
routes/api.phpが作られ、Sanctumも導入される。 


            ./vendor/bin/sail artisan install:api

必要なファイルを作成

Todoモデル、マイグレーション、API用コントローラ、フォームリクエスト、API Resourceを作る。
バリデーションはコントローラではなくフォームリクエストに書く。 


            ./vendor/bin/sail artisan make:model Todo -m
            ./vendor/bin/sail artisan make:controller Api/TodoController --api
            ./vendor/bin/sail artisan make:request StoreTodoRequest
            ./vendor/bin/sail artisan make:request UpdateTodoRequest
            ./vendor/bin/sail artisan make:resource TodoResource

マイグレーションを作成

database/migrations/xxxx_xx_xx_xxxxxx_create_todos_table.phpを書く。


            <?php

            use Illuminate\Database\Migrations\Migration;
            use Illuminate\Database\Schema\Blueprint;
            use Illuminate\Support\Facades\Schema;

            return new class extends Migration
            {
                public function up(): void
                {
                    Schema::create('todos', function (Blueprint $table) {
                        $table->id();
                        $table->string('title');
                        $table->boolean('is_done')->default(false);
                        $table->timestamps();
                    });
                }

                public function down(): void
                {
                    Schema::dropIfExists('todos');
                }
            };

モデルを設定

app/Models/Todo.phpを書く。
fillableは一括登録を許可するカラム、castsは型変換の設定。 


            <?php

            namespace App\Models;

            use Illuminate\Database\Eloquent\Model;

            class Todo extends Model
            {
                protected $fillable = [
                    'title',
                    'is_done',
                ];

                protected $casts = [
                    'is_done' => 'boolean',
                ];
            }

登録用フォームリクエスト

app/Http/Requests/StoreTodoRequest.phpを書く。
APIでも入力チェックはフォームリクエストにまとめる。 


            <?php

            namespace App\Http\Requests;

            use Illuminate\Foundation\Http\FormRequest;

            class StoreTodoRequest extends FormRequest
            {
                public function authorize(): bool
                {
                    return true;
                }

                public function rules(): array
                {
                    return [
                        'title' => ['required', 'string', 'max:255'],
                        'is_done' => ['sometimes', 'boolean'],
                    ];
                }
            }

更新用フォームリクエスト

app/Http/Requests/UpdateTodoRequest.phpを書く。
sometimesは、値が送られてきた時だけその項目をチェックするという意味。 


            <?php

            namespace App\Http\Requests;

            use Illuminate\Foundation\Http\FormRequest;

            class UpdateTodoRequest extends FormRequest
            {
                public function authorize(): bool
                {
                    return true;
                }

                public function rules(): array
                {
                    return [
                        'title' => ['sometimes', 'required', 'string', 'max:255'],
                        'is_done' => ['sometimes', 'boolean'],
                    ];
                }
            }

API Resourceで返却形式を整える

app/Http/Resources/TodoResource.phpを書く。
React側はこのJSONの形を見て画面を作る。 


            <?php

            namespace App\Http\Resources;

            use Illuminate\Http\Request;
            use Illuminate\Http\Resources\Json\JsonResource;

            class TodoResource extends JsonResource
            {
                public function toArray(Request $request): array
                {
                    return [
                        'id' => $this->id,
                        'title' => $this->title,
                        'is_done' => $this->is_done,
                        'created_at' => $this->created_at?->toDateTimeString(),
                        'updated_at' => $this->updated_at?->toDateTimeString(),
                    ];
                }
            }

APIコントローラを作成

app/Http/Controllers/Api/TodoController.phpを書く。
index、store、show、update、destroyでAPIの基本操作を分ける。
$requestや$todoは自分でnewしているのではなく、Laravelがルートやリクエストから渡してくれる。 


            <?php

            namespace App\Http\Controllers\Api;

            use App\Http\Controllers\Controller;
            use App\Http\Requests\StoreTodoRequest;
            use App\Http\Requests\UpdateTodoRequest;
            use App\Http\Resources\TodoResource;
            use App\Models\Todo;

            class TodoController extends Controller
            {
                public function index()
                {
                    return TodoResource::collection(Todo::latest()->get());
                }

                public function store(StoreTodoRequest $request)
                {
                    $todo = Todo::create($request->validated());

                    return (new TodoResource($todo))
                        ->response()
                        ->setStatusCode(201);
                }

                public function show(Todo $todo)
                {
                    return new TodoResource($todo);
                }

                public function update(UpdateTodoRequest $request, Todo $todo)
                {
                    $todo->update($request->validated());

                    return new TodoResource($todo);
                }

                public function destroy(Todo $todo)
                {
                    $todo->delete();

                    return response()->noContent();
                }
            }

コントローラの引数はどこから来るか

Laravelのコントローラメソッドは、ルートに一致した時にLaravelが呼ぶ。
その時、リクエスト情報やURLのidから見つけたModelを引数として渡す。

store(StoreTodoRequest $request) 登録APIに送られた入力値を、Laravelが$requestとして渡す。
StoreTodoRequestなので、rules()のバリデーションを通った値をvalidated()で取り出せる。
show(Todo $todo) URLの {todo} に入ったidを使って、LaravelがTodoを探して$todoとして渡す。
見つからない場合は基本的に404になる。
update(UpdateTodoRequest $request, Todo $todo) Laravelが入力値を$request、URLから見つけたTodoを$todoとして渡す。
そのため、$todo->update($request->validated()) と書ける。
destroy(Todo $todo) 削除対象のTodoをLaravelが探して渡す。
$todo->delete() でその1件を削除する。

APIルートを定義

routes/api.phpを書く。
apiResourceを使うと、一覧、登録、詳細、更新、削除のルートをまとめて定義できる。 


            <?php

            use App\Http\Controllers\Api\TodoController;
            use Illuminate\Support\Facades\Route;

            Route::apiResource('todos', TodoController::class);

DBに反映してルートを確認

マイグレーション後、/api/todosのルートができているか確認する。


            ./vendor/bin/sail artisan migrate
            ./vendor/bin/sail artisan route:list --path=api

APIを確認

ブラウザ、Postman、curlなどで確認する。登録はPOSTなのでJSONを送る。

一覧取得


                curl -i http://localhost/api/todos

登録


                curl -i -X POST http://localhost/api/todos \
                    -H "Accept: application/json" \
                    -H "Content-Type: application/json" \
                    -d '{"title":"買い物に行く"}'

認証が必要なAPI

ログイン済みユーザーだけに見せたいAPIはauth:sanctumを使う。
自分のReact画面から使うSPA認証と、外部アプリから使うBearerトークン認証は考え方が違うので、公式Sanctumを見ながら分けて使う。 


            use Illuminate\Http\Request;
            use Illuminate\Support\Facades\Route;

            Route::middleware('auth:sanctum')->get('/user', function (Request $request) {
                return $request->user();
            });
