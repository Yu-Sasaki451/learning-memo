React 状態管理

stateは、画面に出す値をReactに覚えさせる仕組み。
まずは「仕組みの3行 → コピペ用コード → 値の変化」の順で見る。

まず見るのはこの3行

Reactの入力欄は、ほぼこの3行で考える。
長いコードを見る前に、ここだけ押さえる。

const [title, setTitle] = useState('');
titleという値を作る。最初は空文字。
value={title}
入力欄にtitleを表示する。
onChange={(e) => setTitle(e.target.value)}
入力された文字でtitleを変える。

まずこの4つだけ分かればいい

最初に覚える名前はこれだけ。
ここが分かると、下のコードがかなり読める。

title 今の文字 入力欄に表示したい文字。
setTitle titleを変える関数 titleを変えたい時はこれを呼ぶ。
e 入力イベントの情報 ReactがonChangeの中に渡す。
e.target.value 入力欄の今の文字 ユーザーが打った文字が入る。

コピペ用の完成コード

実際にファイルへ書く時は、この形を使う。
上の3行が、この完成コードの中に入っている。 


            import { useState } from 'react';

            const TodoForm = () => {
                const [title, setTitle] = useState('');

                return (
                    <input
                        type="text"
                        value={title}
                        onChange={(e) => setTitle(e.target.value)}
                    />
                );
            };

            export default TodoForm;

上のコードを日本語にすると

左がコード。右が意味。
ここでは1行ずつではなく、役割ごとに見る。

import { useState } from 'react';
ReactからuseStateを使えるようにする。
const [title, setTitle] = useState('');
titleという値と、titleを変えるsetTitleを作る。最初のtitleは空文字。
value={title}
inputにtitleを表示する。
onChange={(e) => setTitle(e.target.value)}
入力されたら、入力欄の文字をsetTitleに渡してtitleを更新する。

valueとonChangeはセット

stateでinputを管理する時は、この2つをセットで見る。
片方だけだと流れが分かりにくい。

表示する value={title}
+
変える onChange={(e) => setTitle(e.target.value)}

読み方: 「titleを表示して、入力されたらsetTitleでtitleを変える」

入力すると何が変わるか

たとえば「買い物」と入力した場合。
コードの中ではこう動く。

最初 title = ''
入力する e.target.value = '買い物'
stateを変える setTitle('買い物')
次の表示 value={title} // 入力欄に買い物と出る

なぜ (e) => が必要か

onChangeには「入力された時に動かす関数」を渡す。
だから、その場でsetTitleを実行するのではなく、あとで動く関数の形にする。

onChange={setTitle(e.target.value)}
これはダメ。画面を作る時にすぐsetTitleを実行しようとしてしまう。
onChange={(e) => setTitle(e.target.value)}
これはOK。入力された時にReactがeを渡し、その時にsetTitleを実行する。

読み方: 「入力されたら、eを受け取って、入力欄の文字をsetTitleに渡す」

配列に追加する完成コード

Todoを増やす時は、今のtodosを直接変えない。
新しい配列を作ってsetTodosに渡す。 


            const [todos, setTodos] = useState([]);

            const addTodo = (newTodo) => {
                setTodos([...todos, newTodo]);
            };
今のtodos [掃除]
追加するTodo newTodo = 買い物
新しい配列を作る [...todos, newTodo]
次のtodos [掃除, 買い物]

...todos は、今のtodosの中身を新しい配列へコピーする書き方。

配列から削除する完成コード

削除は「消す」より「残す」で考える。
filterで、削除したいid以外だけを残す。 


            const deleteTodo = (id) => {
                const nextTodos = todos.filter((todo) => todo.id !== id);

                setTodos(nextTodos);
            };
今のtodos [id:1 掃除, id:2 買い物, id:3 勉強]
削除したいid id = 2
残す条件 todo.id !== id
次のtodos [id:1 掃除, id:3 勉強]

!== は「同じではない」という意味。

idはどこから渡すか

deleteTodoのidは勝手に入らない。
削除ボタンを押した時に、自分でtodo.idを渡す。 


            <ul>
                {todos.map((todo) => (
                    <li key={todo.id}>
                        {todo.title}
                        <button type="button" onClick={() => deleteTodo(todo.id)}>
                            削除
                        </button>
                    </li>
                ))}
            </ul>
1
mapが1件ずつ渡す todos.map((todo) => ...) todoには、todosの中の1件分が入る。
2
クリックした時にidを渡す onClick={() => deleteTodo(todo.id)} このボタンのTodoのidを、deleteTodoに渡す。

前のstateを使って更新する

今のstateを元に次のstateを作る時は、この形もよく使う。
prevTodosはReactが渡す「更新前のtodos」。 


            setTodos((prevTodos) => {
                return [newTodo, ...prevTodos];
            });
Reactが渡す prevTodos = 今のtodos
先頭に追加 [newTodo, ...prevTodos]
次のtodos 新しいTodoが先頭に来る

公式で確認したこと

React公式では、useStateは現在の値と更新用の関数を返す。
配列やオブジェクトのstateは、直接変えずに新しい値を作って更新する。
