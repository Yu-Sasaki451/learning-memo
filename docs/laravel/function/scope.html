スコープ

Laravelにおけるスコープとは

グローバルスコープは自動で常に動くもの。
ローカルスコープは呼び出した時だけ動くもの。
まずこれがLaravelでの前提。

ローカルスコープはどうして使う

DBからデータを取得するとき、毎回whereを書くとRead、Update、Deleteで同じ条件式が増える。
ローカルスコープにしておくと、条件をモデルにまとめて使い回せる。

検索で使う場合

データを2択にできるものはローカルスコープにしやすい。
例：完了・未完了、公開・非公開、入力済み・未入力。
期間検索のように条件が複雑なものは、別の検索条件として扱うことが多い。

ローカルスコープの型

基本の型があるわけですよ
scope + 名前
引数には必ず$query
リターンで$query を必ず返す

モデルに書くコード


            public function scopeCompleted($query)
            {
                return $query->where('is_completed', true);
            }

コントローラなどで呼び出すコード


            Todo::completed()->get();

特定ユーザーのデータを扱う場合

モデルに書くコード


        public function scopeForUser($query, $userId)
        {
            return $query->where('user_id', $userId);
        }

コントローラなどで呼び出すコード


        Todo::forUser(auth()->id())->get();

入力済み・未入力の２択パターン

モデルに書くコード


                public function scopeFilled($query){
                    return $query->whereNotNull('content');
                }

                public function scopeEmpty($query){
                    return $query->whereNull('content');
                }

コントローラなどで呼び出すコード


                Todo::forUser(auth()->id())
                    ->filled()
                    ->get();
