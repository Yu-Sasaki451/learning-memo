リレーション

1:1,1:N,N:Nのどれなのかをまず考える

1:1は、ユーザーとプロフィールのように片方に対して1つだけ紐づく関係。
1:Nは、カテゴリーとTodoのように1つの親に複数の子が紐づく関係。
N:Nは、ギターと曲のように両方が複数ずつ紐づく関係。

外部キー

1:NではN側が外部キーを持つ。
1:1では、単体で成立しにくい従属側が外部キーを持つことが多い。
N:Nでは元の2つのテーブル同士は外部キーを持たず、中間テーブルに両方の外部キーを持たせる。

1:Nのリレーション

カテゴリーが1側、TodoがN側の例。外部キーはN側のtodosテーブルに追加する。

マイグレーション

$table->foreignId('category_id')->constrained()->cascadeOnDelete();

Categoryモデル

public function todos()
{
    return $this->hasMany(Todo::class);
}

Todoモデル

public function category()
{
    return $this->belongsTo(Category::class);
}

コントローラのuse

use Illuminate\Http\Request;
use App\Models\Todo;
use App\Models\Category;
use App\Http\Requests\TodoRequest;

1:1のリレーション

ユーザーとプロフィールの例。従属するprofilesテーブルにuser_idを持たせる。

マイグレーション

$table->foreignId('user_id')->unique()->constrained()->cascadeOnDelete();

Userモデル

public function profile()
{
    return $this->hasOne(Profile::class);
}

Profileモデル

public function user()
{
    return $this->belongsTo(User::class);
}

コントローラのuse

use Illuminate\Http\Request;
use App\Models\User;
use App\Models\Profile;
use App\Http\Requests\ProfileRequest;

N:Nのリレーション

PostとTagの例。中間テーブルに両方の外部キーを持たせる。

中間テーブルのマイグレーション

Schema::create('post_tag', function (Blueprint $table) {
    $table->foreignId('post_id')->constrained()->cascadeOnDelete();
    $table->foreignId('tag_id')->constrained()->cascadeOnDelete();
    $table->primary(['post_id', 'tag_id']);
});

Postモデル

public function tags()
{
    return $this->belongsToMany(Tag::class);
}

Tagモデル

public function posts()
{
    return $this->belongsToMany(Post::class);
}
