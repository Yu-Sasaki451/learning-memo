URLから処理を追う

画面で起きたことを、URL、routes、Controller、Model、Viewの順に追うためのページです。
CakePHP 2.0のコードリーディングでは、最初にこの流れを作ると迷いにくいです。

CakePHP2特有のURL設定

URLとControllerがそのまま一致しない時は、app/Config/routes.php を見ます。
ここに Router::connect() があると、見た目のURLと実際に呼ぶControllerを変えています。

Router::connect(...) 日本語にすると「このURLが来たら、このControllerとactionに流す」という設定。
':id' URLの一部を変数として受け取る場所。/member/profile/5 の5がidになる。
'controller' => 'users' 実際に呼ぶController。UsersController を指す。
'action' => 'profile' 実際に呼ぶメソッド。UsersController::profile() を指す。
'pass' => array('id') URLから取ったidを、actionの引数へ渡す設定。

コードを読む時の変換

CakePHP2では、URLやファイル名の規約から参照先が決まります。
まず「このURLは、どのController、どのaction、どのViewに行くのか」を日本語に直します。

例

/users/edit/5 は「UsersControllerのeditを呼び、5番のUserを編集する」です。

参照先は app/Controller/UsersController.php、app/Model/User.php、app/View/Users/edit.ctp です。

読む順番

1. URL /users/edit/5 なら、UsersControllerのedit actionを疑う。
2. routes app/Config/routes.php に特別なURL設定がないか見る。
3. Controller app/Controller/UsersController.php の edit() を読む。
4. Model app/Model/User.php の検索、保存、バリデーション、関連を見る。
5. View app/View/Users/edit.ctp のフォーム、リンク、表示変数を見る。

標準URLの読み方

CakePHP 2.0は、特別なroutesがなければURLからControllerとactionを推測できます。
複数形のController、単数形のModelという名前の差に注意します。

日本語に直すと

「usersという画面グループの、editという処理を、id 5に対して実行する」です。

Controllerは複数形の UsersController、Modelは単数形の User になるので、ファイル名検索では両方を探します。

URL /users/edit/5
Controller UsersController。ファイルは app/Controller/UsersController.php。
action edit($id = null)。URLの5が引数に入ることが多い。
Model User。Controller名は複数形、Model名は単数形。
View app/View/Users/edit.ctp。Controller名のフォルダ、action名のctp。

routes.phpに設定がある場合

URLとController名が一致しない時は、routesで別名が付いている可能性があります。
画面のURLでControllerが見つからない時は、最初にroutesを検索します。

日本語に直すと

「ブラウザでは/member/profile/5に見えるが、実際にはusers controllerのprofile actionを呼ぶ。URLのid部分をactionへ渡す」です。

この書き方をする理由は、画面に見せたいURLと、内部のController名を分けたい時があるためです。

Router::connect(
    '/member/profile/:id',
    array('controller' => 'users', 'action' => 'profile'),
    array('pass' => array('id'))
);
/member/profile/5 ブラウザで見えるURL。
'controller' => 'users' UsersController を呼ぶ。
'action' => 'profile' profile() を呼ぶ。
'pass' => array('id') URL内のidをactionの引数へ渡す。

ControllerからViewへ値を渡す

ControllerでDBから取得した値は、$this->set() でViewへ渡します。
View側では、setした名前の変数として使います。

日本語に直すと

「idでUserを1件探す。見つからなければ404。見つかったらViewへuserという名前で渡す。Viewではuserのnameとemailを表示する」です。

参照の流れは、$this->User が app/Model/User.php、$this->set('user', $user) が app/View/Users/profile.ctp へ渡す、です。

class UsersController extends AppController {
    public function profile($id = null) {
        $user = $this->User->find('first', array(
            'conditions' => array('User.id' => $id)
        ));

        if (empty($user)) {
            throw new NotFoundException('ユーザーが見つかりません');
        }

        $this->set('user', $user);
    }
}
<h2><?php echo h($user['User']['name']); ?></h2>
<p><?php echo h($user['User']['email']); ?></p>
$this->User UsersControllerに対応するUserモデル。DB操作を担当する。
find('first') 条件に合うデータを1件取得する。
empty($user) 取得できなかった時の判定。見つからないなら404にする。
$this->set('user', $user) Viewで $user として使えるようにする。
h() HTMLとして危険な文字をそのまま出さないためのエスケープ。

Laravelなら近い形

CakePHP 2.0ではControllerのaction名とViewファイル名が強く結びつきます。
Laravelではroutesに明示して、Controllerからviewをreturnする形がよく使われます。

読み替え

CakePHP2の $this->set('user', $user) は、Laravelの return view('users.profile', ['user' => $user]) に近いです。

違いは、CakePHP2ではactionが終わると規約でctpが表示される点です。LaravelはController内でviewを明示してreturnすることが多いです。

Route::get('/member/profile/{id}', [UserController::class, 'profile']);
public function profile($id)
{
    $user = User::findOrFail($id);

    return view('users.profile', ['user' => $user]);
}
CakePHP 2.0 URL規約とroutesからController/action/Viewへ進む。
Laravel routes/web.phpを入口にしてControllerとviewを追う。
