Controllerの読み方

CakePHP2のControllerを、コードの形、参照先、日本語訳で読む。
Laravelと見た目が違うところを先に潰す。

調べて採用した読み方

CakePHP2公式は、Blog Tutorialで list / add / edit / delete の小さい実例から説明している。
その形に合わせて、このページも「編集処理」を1つ選び、上から順番に分解する。

公式Controller actionはURLから呼ばれ、$this->set() でViewへ値を渡す。
公式App Class App::uses() はクラス名と置き場所をCakePHPへ教える。
公式Request $this->request はリクエスト情報の入口。POST判定や入力値を読む。
公式Saving Data save($this->request->data) はフォーム入力をModel経由で保存する基本形。

まずこの1本を読む

この処理は、ユーザー編集画面のController例です。
URL、親Controller、Model、View、フォーム入力、保存、リダイレクトが全部入っているので、Controller読解の土台になります。 

App::uses('AppController', 'Controller');

class UsersController extends AppController {
    public $components = array('Session');
    public $helpers = array('Html', 'Form');

    public function edit($id = null) {
        if (!$this->User->exists($id)) {
            throw new NotFoundException('ユーザーが見つかりません');
        }

        if ($this->request->is(array('post', 'put'))) {
            if ($this->User->save($this->request->data)) {
                $this->Session->setFlash('ユーザーを更新しました');
                return $this->redirect(array('action' => 'index'));
            }

            $this->Session->setFlash('入力内容を確認してください');
        } else {
            $this->request->data = $this->User->find('first', array(
                'conditions' => array('User.id' => $id)
            ));
        }
    }
}
日本語に直すと

「UsersControllerのeditが呼ばれたら、まずidのUserが存在するか確認する。フォーム送信なら入力値を保存する。保存できたら一覧へ戻す。まだフォーム送信ではない初回表示なら、DBから既存データを取ってフォームへ入れる」です。

1. 読み込みと継承

この処理は、Controller本体が動く前の準備です。
画面の保存や表示の処理ではなく、「このControllerは何を土台にしているか」を示します。 

App::uses('AppController', 'Controller');

class UsersController extends AppController {
App::uses('AppController', 'Controller'); 「AppControllerはController系の場所にある」とCakePHPへ教える。参照先は主に app/Controller/AppController.php。
App CakePHP2のクラス読み込み管理。LaravelのComposer autoloadや use 文より古い書き方。
:: クラス自体の機能を呼ぶ記号。App::uses() は「Appクラスのuses機能を呼ぶ」。
class UsersController UsersControllerというControllerクラスを定義する。ファイルは app/Controller/UsersController.php。
extends AppController AppControllerを親にする。ログイン判定、共通Component、共通設定は親側にあることが多い。
Laravelなら use App\Http\Controllers\Controller; と class UserController extends Controller に近い。

2. ComponentとHelper

この処理は、ControllerとViewで使う道具を指定しています。
会社コードではここに Auth、Paginator、RequestHandler などが増えることがあります。 

public $components = array('Session');
public $helpers = array('Html', 'Form');
public $components Controller内で使う部品。$this->Session のように呼べる。
Session 一時メッセージやセッション情報を扱う。ここでは setFlash() のために使う。
public $helpers Viewで使う部品。ctpで $this->Html、$this->Form として使える。
array('Html', 'Form') PHPの配列。CakePHP2は設定をこの形で渡すことが多い。Laravelの ['Html', 'Form'] に近い。

3. actionとURL

この処理は、ブラウザから呼ばれる入口です。
public function edit($id = null) は「編集画面の処理」という意味で、URLのidを受け取ります。 

public function edit($id = null) {
public function URLから呼べるaction。CakePHP2ではController内のpublicメソッドがactionになる。
edit action名。標準ルールなら /users/edit/5 で呼ばれる。
$id = null URL末尾の5などが入る。来なかった時はnullにしておく、という初期値。
参照するView 通常は app/View/Users/edit.ctp。action名とctp名が対応する。
Laravelなら public function edit($id) と routes/web.php の Route::get('/users/{id}/edit', ...) に近い。

4. 対象データの存在確認

この処理は、不正なidや存在しないidで編集画面を開かれた時に止めるためのものです。
! は否定なので、「存在しないなら」と読みます。 

if (!$this->User->exists($id)) {
    throw new NotFoundException('ユーザーが見つかりません');
}
$this 今のUsersController自身。
-> 左側が持っているものを呼ぶ記号。$this->User は「このControllerが持つUserモデル」。
$this->User 参照先は app/Model/User.php。Controller名Usersに対応する主Model。
exists($id) そのidのUserが存在するか確認する。
throw new NotFoundException 404系の例外を出す。以降の保存処理には進まない。

5. 送信された時だけ保存する

この処理は、編集画面を開いただけなのか、保存ボタンを押した後なのかを分けています。
保存ボタンを押すとPOSTまたはPUTとして来るので、その時だけ save() します。 

if ($this->request->is(array('post', 'put'))) {
    if ($this->User->save($this->request->data)) {
        $this->Session->setFlash('ユーザーを更新しました');
        return $this->redirect(array('action' => 'index'));
    }

    $this->Session->setFlash('入力内容を確認してください');
}
$this->request ブラウザから来たリクエスト情報。POSTか、入力値は何か、URLパラメータは何かを持つ。
is(array('post', 'put')) POSTまたはPUTで送られてきたかを確認する。編集フォームの保存時だけtrueになる。
$this->request->data フォーム入力値。参照元は app/View/Users/edit.ctp の Form->input()。
save(...) Userモデルで保存する。Modelの $validate や beforeSave() が関係する。
setFlash(...) 次の画面に出す一時メッセージをセットする。表示場所はViewやLayoutの Session->flash()。
redirect(array('action' => 'index')) 同じControllerのindexへ移動する。日本語では「保存できたら一覧へ戻る」。

6. 初回表示ではフォームに既存値を入れる

この処理は、まだ保存ボタンを押していない時に動きます。
DBから既存データを取り、$this->request->data に入れることで、FormHelperが入力欄へ初期値を表示します。 

} else {
    $this->request->data = $this->User->find('first', array(
        'conditions' => array('User.id' => $id)
    ));
}
else POSTでもPUTでもない時。つまり編集画面を最初に開いた時。
find('first') 条件に合うUserを1件取得する。
'conditions' 検索条件。SQLのWHEREに近い。
'User.id' => $id 「Userモデルのidが、URLから来たidと同じもの」。=> は左の項目に右の値を対応させる記号。
$this->request->data = ... 取得した既存データをフォームの初期値として置く。Viewの Form->input('name') がこの値を見る。

7. Laravelと見た目が違う場所

同じ「編集保存」でも、Laravelとは書き方が違います。
CakePHP2のコードをLaravelの感覚に置き換えると、次の対応で読むと迷いにくいです。

App::uses(...) Laravelの namespace や use に近いが、CakePHP2では「クラスの置き場所を教える」書き方。
$this->request->data Laravelの $request->all() や $request->validated() に近い入力値。
$this->User->save(...) Laravelの $user->save() や User::create() に近い保存。
$this->set('user', $user) Laravelの return view(..., ['user' => $user]) に近いViewへの値渡し。
$this->Session->setFlash(...) Laravelの with('status', ...) に近い一時メッセージ。
