バリデーション

ルールから逸脱した情報を切り捨てるやつ
フォームリクエスト・条件分岐の2種類やり方があるよ

フォームリクエスト作成

まずはファイルを作る。TodoRequestを作る例。


            php artisan make:request TodoRequest

メソッドを記載

フォームリクエストにメソッドを記載する


            public function authorize(): bool
            {
                return true;
            }

            public function rules(): array
            {
                return [
                    'content' => 'required|string|max:30',
                ];
            }

            public function messages(): array
            {
                return [
                    'content.required' => 'ToDoの内容を入力してください',
                    'content.string' => '文字で入力してください',
                    'content.max' => '入力は30文字以内',
                ];
            }

コントローラの設定もする

useの追加と、メソッドの引数をRequestから作成したフォームリクエストに変更する。

use


            use App\Http\Requests\TodoRequest;

変更前


            public function store(Request $request)

変更後


            public function store(TodoRequest $request)

条件付きバリデーション

新規は必須、更新はNULLでいいという条件とする


                public function rules(): array
                {
                    $rules = [
                        'content' => 'nullable|string|max:20',
                    ];
                
                    if ($this->isMethod('post')) {
                        $rules['content'] = 'required|string|max:20';
                    }

                    return $rules;
                }

ステータスごとの条件バリデーション

ステータスによって変わるという条件つきのやつ


                $rules = [
                    'title' => 'nullable|string|max:50',
                    'status' => 'required|in:draft,published',
                ];

                if ($this->input('status') === 'published') {
                    $rules['title'] = 'required|string|max:50';
                }

ユーザーごとの条件バリデーション

普通のユーザーは20文字制限、管理者は制限なしってパターン


            $rules = [
                'content' => 'required|string',
            ];
            
            if (!$this->user()->is_admin) {
                $rules['content'] .= '|max:20';
            }

データの状態によるバリデーション

完了してるTodoは編集できないってパターン。&&は左も右も両方OKならtrueという意味


            use Illuminate\Validation\Validator;

            public function after(): array
            {
                return [
                    function (Validator $validator) {
                        $todo = Todo::find($this->id);

                        if ($todo && $todo->is_completed) {
                            $validator->errors()->add(
                                'content',
                                '完了済みのToDoは編集できません'
                            );
                        }
                    },
                ];
            }
