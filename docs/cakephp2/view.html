Viewの書き方

Viewは、HTML、表示変数、フォーム、リンク、ページングを見る場所です。
CakePHP 2.0のViewファイルは .ctp で、PHPをそのまま書く形がよく出ます。

Controllerから来た変数を表示する

Controllerで $this->set('user', $user) した値は、Viewで $user として使います。
画面に出す時は、基本的に h() でエスケープします。 

<h2><?php echo h($user['User']['name']); ?></h2>
<p><?php echo h($user['User']['email']); ?></p>
$user Controllerから渡された変数。
['User']['name'] CakePHP2の配列形式。Model名の下にカラム名が入る。
h() HTMLとして危険な文字を無害化して表示する。
echo 画面へ出力する。

foreachで一覧表示

一覧画面では、Controllerから渡された配列を foreach で回します。
CakePHP2では、1件ごとの中にもModel名のキーが入ることを意識します。 

<table>
    <thead>
        <tr>
            <th>ID</th>
            <th>名前</th>
            <th>メール</th>
        </tr>
    </thead>
    <tbody>
        <?php foreach ($users as $user): ?>
            <tr>
                <td><?php echo h($user['User']['id']); ?></td>
                <td><?php echo h($user['User']['name']); ?></td>
                <td><?php echo h($user['User']['email']); ?></td>
            </tr>
        <?php endforeach; ?>
    </tbody>
</table>
foreach 複数件の配列を1件ずつ処理する。
endforeach ViewでHTMLと混ぜやすいPHPの書き方。

ifで表示を切り替える

権限、状態、データ有無によって表示を変える時に使います。
画面にボタンが出ない時は、Viewのif条件とControllerから渡された値を確認します。 

<?php if ($user['User']['status'] === 'active'): ?>
    <span class="status status-active">有効</span>
<?php else: ?>
    <span class="status status-stopped">停止中</span>
<?php endif; ?>
=== 値と型が同じか確認する比較。文字列のactiveと一致するか見ている。
else 条件に当てはまらない時の表示。
endif ViewでHTMLと混ぜやすいPHPのif終了。

FormHelperでフォームを書く

CakePHP2では、フォームを $this->Form で書くことが多いです。
Controllerの $this->request->data とModelのバリデーションにつながります。 

<?php echo $this->Form->create('User'); ?>
<?php echo $this->Form->input('name', array('label' => '名前')); ?>
<?php echo $this->Form->input('email', array('label' => 'メールアドレス')); ?>
<?php echo $this->Form->end('保存'); ?>
Form->create('User') Userモデル用のフォームを開始する。
Form->input('name') name項目の入力欄を作る。バリデーションエラーも出やすい。
label 画面に表示する項目名。
Form->end('保存') 送信ボタンとform終了タグを作る。

リンクと削除ボタン

画面遷移は HtmlHelper、削除のようにPOSTで送りたいものは FormHelper を使うことが多いです。
削除を普通のaタグだけで作っていないか確認します。 

<?php
echo $this->Html->link(
    '編集',
    array('controller' => 'users', 'action' => 'edit', $user['User']['id'])
);
?>

<?php
echo $this->Form->postLink(
    '削除',
    array('action' => 'delete', $user['User']['id']),
    array('confirm' => '本当に削除しますか？')
);
?>
Html->link 通常のリンクを作る。
postLink POST送信するリンク風の操作。削除でよく使う。
confirm 実行前の確認メッセージ。

フラッシュメッセージ

Controllerで Session->setFlash() したメッセージは、ViewやLayout側で表示します。
保存成功メッセージが出ない時は、Layoutに表示処理があるか見ます。 

<?php echo $this->Session->flash(); ?>
Session->flash() 一時メッセージを画面に表示する。
探す場所 app/View/Layouts/default.ctp に書かれていることが多い。

ページング

一覧でページ移動や並び替えがある場合は、ControllerのPaginator設定とViewのPaginatorHelperをセットで見ます。 

<th><?php echo $this->Paginator->sort('User.name', '名前'); ?></th>

<div class="paging">
    <?php echo $this->Paginator->prev('前へ'); ?>
    <?php echo $this->Paginator->numbers(); ?>
    <?php echo $this->Paginator->next('次へ'); ?>
</div>
sort クリックで並び替えできる見出し。
prev / next 前ページ、次ページのリンク。
numbers ページ番号のリンク。

ElementとLayout

ヘッダー、サイドバー、共通パーツは、今見ているctpに直接ないことがあります。
その場合はLayoutやElementを探します。 

<?php echo $this->element('user_menu', array('user' => $user)); ?>
element('user_menu') app/View/Elements/user_menu.ctp を読み込む。
Layouts/default.ctp ページ全体の共通HTML。ヘッダーやフッターがあることが多い。

Laravel Bladeなら近い形

<h2>{{ $user->name }}</h2>

@foreach ($users as $user)
    <p>{{ $user->email }}</p>
@endforeach

<form method="POST" action="{{ route('users.store') }}">
    @csrf
    <input type="text" name="name" value="{{ old('name') }}">
    <button type="submit">保存</button>
</form>
CakePHP 2.0 <?php echo h(...) ?>、FormHelper、HtmlHelper をよく見る。
Laravel {{ ... }}、@foreach、route()、@csrf をよく見る。
