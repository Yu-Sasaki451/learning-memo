CakePHP 2.0 コードリーディング

CakePHP 2.0のコードを読む時に、よく出る書き方を実例で見るページです。
Laravel経験がある場合に対応関係も見られるようにしています。

このページ群で見ること

MVCの概念説明ではなく、実際のコードを読んだ時に迷いやすい部分を日本語へ直して見ます。
どのファイルを参照しているのか、なぜその形で書くのか、Laravelだとどの感覚に近いのかを確認します。

日本語に直す $this->User->save($this->request->data) を「フォーム入力をUserとして保存する」のように読む。
参照先を見る $this->User なら app/Model/User.php、$this->set ならViewの変数を見る。
形の理由を見る array('conditions' => ...) は検索条件をCakePHP2のORMへ渡すための形として読む。

公式で確認したこと

CakePHP 2.0のコードは、CakePHP 2.x公式を基準に見る。
Laravelとの比較は理解補助で、実際の会社コードではCakePHP 2.0側の既存パターンを優先する。

Laravelとのざっくり対応

完全に同じではない。
ただ、読む時の入口としては次の対応で考えると迷いにくい。

CakePHP 2.0 Laravelで近いもの
app/Controller/UsersController.php app/Http/Controllers/UserController.php。画面や保存処理の入口。
app/Model/User.php app/Models/User.php。DB取得、保存、関連、バリデーションを見る。
app/View/Users/edit.ctp resources/views/users/edit.blade.php。HTMLと表示処理。
$this->User->find('first') User::where(...)->first() に近い。
$this->User->save($this->request->data) $user->save() や User::create() に近い。
public $validate FormRequestや $request->validate() に近い。ただしCakePHP 2ではModelに書くことが多い。
$this->set('user', $user) return view(..., ['user' => $user]) に近い。
