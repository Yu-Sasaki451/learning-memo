Reactへ

Laravel Sail 最低限の環境構築

ここは、LaravelをSailで起動してReactを表示できるところまで。
ルーティング、ページ分割、コンポーネント化は次のページに分ける。

Dockerを起動して確認

Docker Desktopを起動してから実行する。ここでエラーなら、先にDocker Desktopを起動する。


            docker info

プロジェクトを作る場所へ移動

例としてデスクトップに作る。別の場所に作るなら、cdの後ろだけ変える。


            cd ~/Desktop

Docker経由でLaravelを作成

この例では、laravel/laravel:^10.0 がLaravelのバージョン指定、todo-react がプロジェクト名。
Laravel 12系で作るなら、laravel/laravel:^12.0 のように変える。 


            docker run --rm \
                -u "$(id -u):$(id -g)" \
                -v "$(pwd):/var/www/html" \
                -w /var/www/html \
                -e COMPOSER_CACHE_DIR=/tmp/composer_cache \
                laravelsail/php82-composer:latest \
                composer create-project laravel/laravel:^10.0 todo-react

作成したプロジェクトへ移動

ここから先は、作成したLaravelプロジェクトの中で実行する。


            cd todo-react

Sailをインストール

docker-compose.ymlを作り、.envにDocker用のDB設定を追加するためのコマンド。


            php artisan sail:install

.envのDB設定を確認

SailのMySQLへ接続するので、DB_HOSTはmysqlにする。


            DB_CONNECTION=mysql
            DB_HOST=mysql
            DB_PORT=3306
            DB_DATABASE=laravel
            DB_USERNAME=sail
            DB_PASSWORD=password

phpMyAdminを追加

docker-compose.ymlのservices内に追加する。networksはsailにする。


            phpmyadmin:
                image: phpmyadmin/phpmyadmin
                platform: linux/amd64
                environment:
                    - PMA_ARBITRARY=1
                    - PMA_HOST=mysql
                    - PMA_USER=${DB_USERNAME}
                    - PMA_PASSWORD=${DB_PASSWORD}
                depends_on:
                    - mysql
                ports:
                    - 8080:80
                networks:
                    - sail

Sailを起動

-dは、ターミナルを占有せずに裏側で起動するためのオプション。


            ./vendor/bin/sail up -d

画面を確認

Laravelはlocalhost、phpMyAdminは8080番で開く。

Laravel


            http://localhost

phpMyAdmin


            http://localhost:8080

Nodeモジュールをインストール

npmコマンドもSail経由で実行する。


            ./vendor/bin/sail npm install

Reactをインストール


            ./vendor/bin/sail npm install react react-dom

ViteのReactプラグインをインストール

--save-devは、開発時に使うパッケージとして追加するという意味。


            ./vendor/bin/sail npm install @vitejs/plugin-react@4 --save-dev

vite.config.jsをReact用に変更

resources/js/app.jsxをViteの読み込み対象にする。Blade側でも同じパスを書く。


            import { defineConfig } from 'vite';
            import laravel from 'laravel-vite-plugin';
            import react from '@vitejs/plugin-react';

            export default defineConfig({
                plugins: [
                    laravel([
                        'resources/css/app.css',
                        'resources/js/app.jsx',
                    ]),
                    react(),
                ],
            });

React表示用のapp.jsxを作成

resources/js/app.jsxに書く。ここではまずReactが表示できるかだけ確認する。


            import React from 'react';
            import ReactDOM from 'react-dom/client';

            function App() {
                return <h1>Hello React</h1>;
            }

            ReactDOM.createRoot(document.getElementById('app')).render(<App />);

BladeでReactを表示

resources/views/index.blade.phpを作成して書く。@viteReactRefreshは@viteより前に書く。


            <!DOCTYPE html>
            <html lang="ja">
            <head>
                <meta charset="UTF-8">
                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                <title>Todo React</title>
                @viteReactRefresh
                @vite(['resources/css/app.css', 'resources/js/app.jsx'])
            </head>
            <body>
                <div id="app"></div>
            </body>
            </html>

最初の表示用ルート

routes/web.phpに書く。ここでは最低限、トップページでindex.blade.phpを返すだけにする。


            Route::get('/', function () {
                return view('index');
            });

Viteを起動

Reactの画面を編集して確認する間は、このコマンドを動かしておく。


            ./vendor/bin/sail npm run dev

ここまでが最低限

ここまでできれば、LaravelをSailで起動してReactを表示できる。
React Router、URL定義、ページ分割、コンポーネント化はReactカテゴリで進める。
