Modelの書き方

Modelは、DB取得、保存、関連、バリデーションを見る場所です。
CakePHP 2.0では、入力チェックがControllerではなくModelに書かれていることがよくあります。

CakePHP2特有の書き方

Modelの先頭や設定配列は、LaravelのEloquentと見た目が違います。
ここでは「何を参照しているのか」を先に確認します。

App::uses('AppModel', 'Model'); 日本語にすると「AppModelクラスはModelフォルダ側にあるので、使う時に読めるようCakePHPへ場所を教える」です。
class User extends AppModel 「AppModelを土台にしてUserモデルを作る」。共通のDB処理やコールバックはAppModel側にあることがある。
public $useTable = 'users'; このModelが参照するDBテーブル名。Laravelの protected $table = 'users'; に近い。
public $validate = array(...); 保存時の入力チェック。LaravelのFormRequestに近い役割だが、CakePHP2ではModelに置かれていることが多い。
'User.id' => $id 「Userテーブルのidカラムが、変数idと同じもの」という条件。SQLのWHEREに近い。
このページのコードを読む時の結論

Modelでは、まず どのテーブル、どの条件、保存前にどのルール を見ているかを読む。

Modelコードの読み替え

Modelでは「どのテーブルを見るか」「どの条件で取るか」「保存前に何を確認するか」を読みます。
SQLを直接書かず、CakePHPの配列で条件を書くことが多いです。

'conditions' SQLのWHEREに近い。「この条件に合う行だけ取る」という意味。
'Model.field' どのModelのどのカラムかを表す。'User.email' はUserモデルのemailカラム。
'fields' SELECTするカラム。全部ではなく必要な項目だけ取る時に使う。
'order' ORDER BYに近い。表示順、取得順を決める。
'recursive' 関連データをどこまで取るか。読みにくい配列や重い画面の原因になることがある。

Modelの基本形

Model名は単数形です。
User モデルは、通常 users テーブルを扱います。

日本語に直すと

「UserというModelを作る。このModelはusersテーブルを使い、idを主キーにして、代表表示名はnameにする」です。

参照している先はDBの users テーブルです。Controllerからは $this->User として呼ばれます。

App::uses('AppModel', 'Model');

class User extends AppModel {
    public $useTable = 'users';
    public $primaryKey = 'id';
    public $displayField = 'name';
}
extends AppModel 共通Modelを継承する。全Model共通の処理がAppModelにあることが多い。
AppModel 参照先は app/Model/AppModel.php。全Model共通の親クラス。
$useTable 使うテーブル名を指定する。規約通りなら省略されていることもある。
$primaryKey 主キー。通常はid。
$displayField 一覧やselectで代表名として使う項目。

関連 belongsTo / hasMany

関連は、別テーブルとつながる設定です。
コードリーディングでは、取得結果の配列に別Modelのデータが混ざる理由をここで確認します。

日本語に直すと

「PostはUserに属している。PostはCommentを複数持っている。Postを取る時にUserやCommentが一緒に付いてくる可能性がある」です。

参照している先は users.id、posts.user_id、comments.post_id のような外部キーです。

class Post extends AppModel {
    public $belongsTo = array(
        'User' => array(
            'className' => 'User',
            'foreignKey' => 'user_id'
        )
    );

    public $hasMany = array(
        'Comment' => array(
            'className' => 'Comment',
            'foreignKey' => 'post_id',
            'dependent' => true
        )
    );
}
belongsTo このModelが親を持つ。PostはUserに所属する。DB上は posts.user_id から users.id を見る。
hasMany このModelが複数の子を持つ。PostはCommentを複数持つ。DB上は comments.post_id からPostを見に来る。
foreignKey 関連に使う外部キー。user_id、post_id など。
dependent 親を削除した時に子も削除するか。影響が大きいので注意して読む。

バリデーション

CakePHP 2.0では、Modelの $validate に入力チェックが書かれていることがよくあります。
保存できない時は、ControllerだけでなくModelのバリデーションを必ず見ます。

日本語に直すと

「nameは空欄禁止で50文字以内。emailはメールアドレス形式でないと保存しない。違反したらmessageの文章を画面へ出す」です。

参照元はControllerの $this->User->save($this->request->data) です。save時にこのルールが使われます。

public $validate = array(
    'name' => array(
        'notBlank' => array(
            'rule' => 'notBlank',
            'message' => '名前を入力してください'
        ),
        'maxLength' => array(
            'rule' => array('maxLength', 50),
            'message' => '名前は50文字以内で入力してください'
        )
    ),
    'email' => array(
        'email' => array(
            'rule' => 'email',
            'message' => 'メールアドレスの形式で入力してください'
        )
    )
);
notBlank 空欄を許可しない。'rule' => 'notBlank' は「この項目は空ではだめ」という指定。
maxLength 文字数の上限を決める。array('maxLength', 50) は「50文字まで」という指定。
email メールアドレス形式か確認する。
message 入力エラー時に画面へ出る文言。

findのよく使う形

CakePHP 2.0のDB取得では、find() が頻繁に出ます。
まず、何件取るのか、どんな条件か、どの項目を取るのかを見ます。

日本語に直すと

find('first') は「1件だけ探す」、find('all') は「複数件探す」、find('count') は「件数だけ数える」、find('list') は「select用のidと名前の一覧を作る」です。

'User.status' => 'active' は「usersテーブルのstatusがactiveのもの」という条件です。

$user = $this->User->find('first', array(
    'conditions' => array('User.id' => $id),
    'recursive' => -1
));

$users = $this->User->find('all', array(
    'conditions' => array('User.status' => 'active'),
    'fields' => array('User.id', 'User.name', 'User.email'),
    'order' => array('User.created' => 'desc'),
    'limit' => 20
));

$count = $this->User->find('count', array(
    'conditions' => array('User.status' => 'active')
));

$list = $this->User->find('list', array(
    'fields' => array('User.id', 'User.name')
));
first 1件取得する。
all 複数件取得する。
count 件数だけ取得する。
list select用のidと表示名の配列を作る時によく使う。
recursive 関連データをどこまで取るか。重い画面ではここが原因になることがある。

保存 create / save / saveAll

保存できない時は、送られてきた配列の形とModelのバリデーションを見ます。
CakePHP2では配列のキーにModel名が入る形がよく出ます。

日本語に直すと

「Userのnameとemailを保存する。新規登録なので先にcreateで新しい行として扱う準備をする」です。

CakePHP2の保存データは array('User' => array(...)) の形が基本です。FormHelperで作ったフォームもこの形で送られます。

$data = array(
    'User' => array(
        'name' => '山田太郎',
        'email' => 'yamada@example.com'
    )
);

$this->User->create();
$this->User->save($data);
$data = array(
    'User' => array(
        'name' => '山田太郎',
        'email' => 'yamada@example.com'
    ),
    'Profile' => array(
        'birthday' => '1990-01-01'
    )
);

$this->User->saveAll($data);
create() 新規登録として保存する準備。
save() 1つのModelを保存する。
saveAll() 関連する複数Modelをまとめて保存する。

Model内に処理をまとめる

Controllerが長すぎる時は、検索条件や業務ルールがModelのメソッドに分けられていることがあります。
$this->User->findActiveByEmail() のような呼び出しを見たら、Model内の同名メソッドを探します。

日本語に直すと

「渡されたメールアドレスで、有効状態のUserを1件探して返す」です。

Controllerで $this->User->findActiveByEmail($email) と書かれていたら、このModelメソッドが参照先です。

public function findActiveByEmail($email) {
    return $this->find('first', array(
        'conditions' => array(
            'User.email' => $email,
            'User.status' => 'active'
        ),
        'recursive' => -1
    ));
}
この処理は 有効なユーザーをメールアドレスで1件探す処理。
読む時の注意 Controllerだけ見ても条件が分からない。Modelのメソッド定義まで追う。

beforeSave

保存前に値が変わる場合は、Modelのコールバックを確認します。
画面では入力していない値がDBに入る時は、beforeSave() を探します。

日本語に直すと

「保存直前にemailがあれば、小文字に変換してから保存を続ける」です。

参照元は save() です。Controllerからsaveを呼ぶと、保存前に自動でbeforeSaveが動きます。

public function beforeSave($options = array()) {
    if (isset($this->data[$this->alias]['email'])) {
        $this->data[$this->alias]['email'] = strtolower($this->data[$this->alias]['email']);
    }

    return true;
}
beforeSave saveの直前に自動で呼ばれる。
$this->data 保存対象のデータ。
return true 保存を続ける。falseを返すと保存が止まる。

Laravelなら近い形

class User extends Model
{
    protected $fillable = ['name', 'email'];

    public function posts()
    {
        return $this->hasMany(Post::class);
    }
}
CakePHP 2.0 Modelにfind、save、関連、validate、beforeSaveがまとまりやすい。
Laravel Eloquent Modelに関連やfillableを書き、入力チェックはFormRequest側に分けることが多い。
