コマンドライン

コマンドは「何のツールで」「何をするか」「対象は何か」を左から読む。
コピペする時は、実行する場所も一緒に確認する。

公式で確認したこと

Docker Composeはcompose.ymlがあるプロジェクトでコンテナを操作する。
Laravelのartisanは、コントローラ作成、マイグレーション実行、シーダー実行などをコマンドで行うための仕組み。

  • Docker公式 docker compose up
  • Laravel公式 Artisan Console
  • Laravel公式 Migrations

コマンドの読み方

コマンドはスペースで区切って読む。
たとえば、php artisan make:controller TodoController は4つのかたまりに分かれる。

php PHPで実行する、という意味。
artisan Laravelのコマンドを動かすためのファイル。
make:controller コントローラを作る命令。
TodoController 作成するコントローラの名前。ここは作りたい名前に変える。

今いる場所を確認する

pwdは、今ターミナルが開いている場所を表示する。
Laravelのコマンドは、基本的にLaravelプロジェクトの直下で実行する。 

pwd

Dockerコンテナを起動する

compose.ymlがあるディレクトリで実行する。
-dはバックグラウンドで起動するオプション。 

docker compose up -d
docker Dockerを操作するコマンド。
compose 複数のコンテナをまとめて扱う。
up コンテナを作成して起動する。
-d ターミナルを占有せず、裏側で起動する。

Dockerコンテナを停止する

コンテナを止めるだけならstopを使う。
データを消すコマンドではないので、開発中に一時停止したい時に使う。 

docker compose stop

PHPコンテナに入る

Docker環境でPHPやComposerを直接触りたい時に使う。
phpはcompose.ymlに定義しているサービス名なので、プロジェクトによって名前が違うことがある。 

docker compose exec php bash
exec 起動中のコンテナの中でコマンドを実行する。
php 入る対象のサービス名。compose.yml側の名前と合わせる。
bash コンテナの中でコマンドを打てる状態にする。

コントローラを作成する

TodoControllerを作る例。
作成後は app/Http/Controllers の中にファイルができる。 

php artisan make:controller TodoController

モデルを作成する

Todoモデルを作る例。
作成後は app/Models の中にファイルができる。 

php artisan make:model Todo

フォームリクエストを作成する

バリデーション専用のクラスを作る。
このメモでは、バリデーションは基本的にコントローラではなくフォームリクエストに書く。 

php artisan make:request TodoRequest

マイグレーションを作成する

todosテーブルを作るためのマイグレーションファイルを作成する。
作成後は database/migrations の中にファイルができる。 

php artisan make:migration create_todos_table

マイグレーションを実行する

database/migrations の内容をDBに反映する。
カラムを書いた後に実行する。 

php artisan migrate

テーブルを作り直してマイグレーションする

migrate:freshは全テーブルを削除してから作り直す。
本番環境や残したいデータがあるDBでは使わない。 

php artisan migrate:fresh

シーダーを作成する

ダミーデータを入れるためのファイルを作る。
作成後は database/seeders の中にファイルができる。 

php artisan make:seeder TodoSeeder

シーディングを実行する

シーダーに書いたダミーデータをDBに入れる。
どのシーダーを呼ぶかは DatabaseSeeder.php 側も確認する。 

php artisan db:seed
