ミドルウェア

どう使うのか

ミドルウェアは、コントローラの前段でリクエストを通して良いか判断する仕組み。
ログインチェックや権限チェックなどで使う。
Laravel 11以降は、登録が必要ならbootstrap/app.phpに書く。

ミドルウェアの雛形

処理前にチェックして、問題なければreturn $next($request);で次へ進める。

public function handle(Request $request, Closure $next)
{
    $response = $next($request);

    return $response;
}

管理者だけを通すミドルウェア

!は否定の意味。
||はORの意味で、左か右のどちらか一方でも当てはまればtrueになる。
つまり、ログインしていない、または管理者ではない場合は403エラーにする。

ミドルウェア本体

public function handle(Request $request, Closure $next)
{
    if (!auth()->user() || !auth()->user()->is_admin) {
        abort(403);
    }

    return $next($request);
}

bootstrap/app.phpに登録

use App\Http\Middleware\AdminOnly;
use Illuminate\Foundation\Configuration\Middleware;

->withMiddleware(function (Middleware $middleware) {
    $middleware->alias([
        'admin' => AdminOnly::class,
    ]);
})

routes/web.php

Route::middleware('admin')->group(function () {
    Route::get('/admin', [AdminController::class, 'index']);
});
