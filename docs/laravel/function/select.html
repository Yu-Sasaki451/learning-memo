ドロップダウンとセレクト

DBを参照するパターン

DBの中身が変われば反映されるので、変化に強い。前提としてコントローラから$categoriesを渡しておく。

<select name="category_id">
    @foreach ($categories as $category)
        <option value="{{ $category->id }}">
            {{ $category->name }}
        </option>
    @endforeach
</select>

手書きするパターン

選択肢がほぼ変わらない場合は、Bladeに直接書いてもよい。

<select name="status">
    <option value="draft">下書き</option>
    <option value="published">公開</option>
</select>

定数クラス式 実務向き

app/Constants/TodoStatus.phpを作る。

app/Constants/TodoStatus.php

<?php

namespace App\Constants;

class TodoStatus
{
    public const LIST = [
        'pending' => '未完了',
        'completed' => '完了',
    ];
}

バリデーションルール

use App\Constants\TodoStatus;

'status' => 'required|in:' . implode(',', array_keys(TodoStatus::LIST)),

Blade

<select name="status">
    @foreach (\App\Constants\TodoStatus::LIST as $key => $label)
        <option value="{{ $key }}">{{ $label }}</option>
    @endforeach
</select>

config式 スピード重視

config/todo.phpを使う。

config/todo.php

<?php

return [
    'status' => [
        'pending' => '未完了',
        'completed' => '完了',
    ],
];

Blade

<select name="status">
    @foreach (config('todo.status') as $key => $label)
        <option value="{{ $key }}">{{ $label }}</option>
    @endforeach
</select>

Enum式

app/Enums/TodoStatus.phpを作る。PHP 8.1以降で使える。

app/Enums/TodoStatus.php

<?php

namespace App\Enums;

enum TodoStatus: string
{
    case Pending = 'pending';
    case Completed = 'completed';

    public function label(): string
    {
        return match ($this) {
            self::Pending => '未完了',
            self::Completed => '完了',
        };
    }
}

Blade

<select name="status">
    @foreach (\App\Enums\TodoStatus::cases() as $status)
        <option value="{{ $status->value }}">
            {{ $status->label() }}
        </option>
    @endforeach
</select>
