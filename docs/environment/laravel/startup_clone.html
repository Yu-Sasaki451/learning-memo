クローンして環境構築

まずGitHubでリポジトリ作成

ブラウザでGitHubを開いて、空のリポジトリを作る。
README、.gitignore、licenseはこの時点では追加しない。

プロジェクトを作成したいディレクトリへ

例としてデスクトップに移動する。別の場所に置きたい場合は、cdの後ろのパスだけ変える。


            cd ~/Desktop

自分用の値を設定

変数は、ターミナル内で一時的に値を覚える名前。
TEMPLATE_REPOSITORY_URLは複製元、OWN_REPOSITORY_URLは自分のGitHubリポジトリのSSH URLに変える。 


            PROJECT_NAME="laravel-practice"
            TEMPLATE_REPOSITORY_URL="git@github.com:your-name/laravel-docker-template.git"
            OWN_REPOSITORY_URL="git@github.com:your-name/laravel-practice.git"

クローン実行

複製元リポジトリを、PROJECT_NAMEで指定した名前のディレクトリとして作る。


            git clone "$TEMPLATE_REPOSITORY_URL" "$PROJECT_NAME"

プロジェクトディレクトリに移動

以降のgitコマンドやdocker composeコマンドは、プロジェクトディレクトリ内で実行する。


            cd "$PROJECT_NAME"

リモートリポジトリ変更

今のままだとただ複製しただけなので権利を変更する
自分の作成したリポジトリに紐付け作業をする
OWN_REPOSITORY_URLに入れた自分のGitHubリポジトリへ向ける 


            git remote set-url origin "$OWN_REPOSITORY_URL"

紐付けできたか確認する


            git remote -v

addしてコミットしてpush

まとめて下記に書くけど1行ずつ実行すること


            git add .
            git commit -m "リモートリポジトリ変更"
            git push origin main

ymlを確認してコンテナビルド

一旦ビルドしてみるけど
エラーになるようならymlの中身を編集して再ビルド 


            docker compose up -d --build

コンポーザーをインストール

割愛するけどphpコンテナに入っておくこと


            composer install

.env複製＆編集

下記のコードで.envを作成する
exampleってなってることあるから.envをちゃんと作る
作ったらDBのとこをymlのmysqlに沿って編集する 


            cp .env.example .env

キーの設定

APP_KEYを作るために必須なので実行する。 


            php artisan key:generate

コンフィグクリア


            php artisan config:clear

キャッシュクリア


            php artisan cache:clear
