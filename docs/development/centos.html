CentOSとは何か

CakePHP 2.0の開発で出てくるCentOSの役割、通常のローカル開発との違い、確認コマンドの使い方をまとめたページです。
まず「自分のPCで作業しているのか」「CentOSサーバーで作業しているのか」を分けて考える。

公式で確認する情報

CentOSはLinux系のOSで、会社のWebアプリを動かすサーバーとして使われることがある。
古いCentOSはサポート終了しているものがあるので、まずバージョン確認が必要。

  • CentOS公式 CentOS Linux EOL
  • VS Code公式 Remote SSH
  • VS Code公式 Terminal Basics
  • Apache公式 Log Files
  • Apache公式 apachectl
  • systemctl man page

CentOSの役割

CentOSは、CakePHPのコードを動かす土台になるOS。
ブラウザからアクセスが来ると、Apacheが受け取り、PHPがCakePHPのコードを実行し、必要ならDBへアクセスする。

自分のPC VS Code、ブラウザ、チャットを開いて操作する端末。MacやWindowsのことが多い。
CentOSサーバー PHP、Apache、CakePHP、ログ、DB接続設定がある場所。VS Code Remote SSHで入って作業することがある。
Apache ブラウザから来たHTTPアクセスを受け取るWebサーバー。CentOSではサービス名が httpd のことが多い。
PHP CakePHP 2.0のコードを実行する。サーバー側に入っているPHPバージョンで動く。
CakePHP PHPで作られたフレームワーク。Controller、Model、Viewに分けてアプリを書く。
DB ユーザー、商品、注文などのデータを保存する場所。CakePHPのModelから接続する。

通常のローカル開発との違い

通常のローカル開発では、自分のPCにコード、PHP、DB、Webサーバーを入れて動かすことが多い。
CentOS開発では、それらがサーバー側にあり、VS Codeから接続して触ることがある。

コードの場所 ローカル開発: 自分のPC内のフォルダ。
CentOS開発: サーバー上のフォルダをVS Code越しに開いていることがある。
コマンドの実行場所 ローカル開発: 自分のPCで動く。
Remote SSH: CentOSサーバー上で動く。
Webサーバー ローカル開発: PHP内蔵サーバー、Docker、MAMPなどを使うことがある。
CentOS開発: Apacheの httpd が使われることが多い。
ログ ローカル開発: ターミナルやアプリ内ログを見ることが多い。
CentOS開発: app/tmp/logs と /var/log/httpd を分けて見る。
権限 ローカル開発: 自分のユーザーで書き込めることが多い。
CentOS開発: SSHユーザー、Apacheユーザー、ファイル所有者が違うと書き込みに失敗する。
インストール Mac: brew。Ubuntu系: apt。CentOS: yum や dnf。
サーバーへ勝手にインストールせず、会社ルールを確認する。
サービス操作 CentOSでは systemctl でApacheなどの状態を見る。
再起動は他の人に影響するので、確認なしに実行しない。

VS Code Remote SSHで起きていること

Remote SSHは、VS Codeの画面は自分のPCにあるまま、作業先をCentOSサーバーにする仕組み。
つまり、VS Codeで保存したファイルやターミナルで実行したコマンドは、サーバー側に反映される。

1. VS Codeを開く 操作画面は自分のPCにある。
2. SSH接続する VS CodeがCentOSサーバーに接続する。
3. フォルダを開く Explorerに出ているのはサーバー上のプロジェクトフォルダ。
4. ファイルを保存する 保存されるのはサーバー上のファイル。
5. ターミナルを開く コマンドはサーバー上で実行される。

確認: VS Code左下の接続先表示を見る。ターミナルで hostname を実行すると、どのサーバーで作業しているか確認できる。

コマンドを使う流れ

CentOS環境では、いきなり原因のファイルを直すのではなく、まず「どこで」「誰として」「何が動いているか」を確認する。
次の順番で見ると、CakePHPの問題か、サーバーの問題かを分けやすい。 

hostname
whoami
pwd
id
cat /etc/centos-release
which php
php -v
httpd -v
1. どのサーバーか hostname
開発環境、検証環境、本番環境を間違えないために見る。
2. 誰として操作しているか whoami、id
ファイルを書けるか、ログを読めるか、権限の判断に使う。
3. どこで操作しているか pwd
プロジェクト直下なのか、appの中なのかを確認する。
4. OSは何か cat /etc/centos-release
CentOSのバージョンを見る。古い環境では使えるPHPやパッケージが限られる。
5. PHPは何か which php、php -v
どのPHPでCakePHPが動くかを確認する。
6. Apacheは何か httpd -v
Apacheのバージョンを見る。Apacheがない、または別構成なら会社ルールを確認する。

ログを見る流れ

エラーを見る時は、CakePHPのログとApacheのログを分ける。
ControllerやModelのエラーならCakePHPログ、サーバー設定や権限の問題ならApacheログに出ることがある。 

tail -f app/tmp/logs/error.log
tail -f app/tmp/logs/debug.log
sudo tail -f /var/log/httpd/error_log
sudo tail -f /var/log/httpd/access_log
CakePHP error.log app/tmp/logs/error.log
CakePHPの処理中に起きたエラーを見る。
CakePHP debug.log app/tmp/logs/debug.log
開発中に出したdebugログを見る。
Apache error_log /var/log/httpd/error_log
Apache、PHP読み込み、権限、.htaccessなどの問題を見る。
Apache access_log /var/log/httpd/access_log
どのURLへアクセスが来たか、HTTPステータスが何かを見る。

権限を見る流れ

CentOSでは、ファイルの持ち主や権限が原因で、保存、ログ出力、キャッシュ更新に失敗することがある。
CakePHP 2.0では app/tmp にログやキャッシュを書くので、ここを確認する。 

ls -ld app/tmp
ls -ld app/tmp/logs
ls -l app/tmp/logs
whoami
id
何を見るか app/tmp と app/tmp/logs の所有者、グループ、書き込み権限を見る。
よくある症状 ログが出ない、画面が真っ白、キャッシュが更新されない、Permission deniedが出る。
やってはいけないこと 意味が分からないまま chmod 777 を実行しない。セキュリティ事故につながる。
相談する時 lsの結果、whoami、id、出ているエラーをセットで伝える。

Apacheの状態を見る流れ

ブラウザで画面が開けない時は、CakePHPの前にApacheが動いているかを見る。
ただし、再起動や設定変更は他の人に影響するので、確認なしに実行しない。 

sudo systemctl status httpd
sudo apachectl configtest
systemctl status httpd Apacheが起動しているか、失敗していないかを見る。
apachectl configtest Apacheの設定ファイルに文法エラーがないか確認する。
再起動 restart や reload は勝手に実行しない。必要なら先輩や運用担当に確認する。

困った時の切り分け

エラーが出た時は、どの層の問題かを分ける。
これを分けると、どのログを見るか、誰に聞くか、どのファイルを直すかが決めやすい。

画面の文言や入力欄 View、Element、Layoutを見る。
保存や検索 Controller、Model、DB、バリデーションを見る。
真っ白画面や500エラー CakePHPログとApache error_logを見る。
アクセスできない Apache、URL、VirtualHost、hosts、ネットワーク、権限を確認する。
ログが出ない app/tmp/logsの権限、CakePHPのdebug設定、Apacheログを見る。
