TypeScript

JavaScriptに型を足して、実行前にミスを見つけるためのメモ。

まず覚える順番

TypeScriptは新しい別言語というより、JavaScriptに「この値は文字です」「これはTodoの配列です」と説明を足すもの。
先にJavaScriptの基礎が分かっていれば、型の読み方から始めればよい。

title: string : は型を書く titleには文字列が入る。
Todo[] [] は配列型 Todoが複数入る配列。
string | null | は または 文字列かnullのどちらか。
disabled?: boolean ? は省略できる 渡しても渡さなくてもよい。
1. 型チェックの目的 実行する前に、値の使い方の間違いを見つける。
2. よく使う型 string、number、boolean、配列、オブジェクトを読む。
3. 型を絞る nullや複数候補の型を、ifで安全に扱う。
4. Reactで使う props、state、event、APIレスポンスに型を付ける。

このサイト内での役割: TypeScriptはReactのprops、state、APIレスポンスの形を安全にするために使う。 記号が分からない時は コードの記号、 どこにつながるかは 全体の流れ と一緒に見ると分かりやすい。
