JavaScript

JavaScriptを怖くしないために、文法、配列、画面操作、API通信、Reactで出る書き方を分けて整理する。

先に覚える順番

JavaScriptは全部を一気に覚えようとすると難しく見える。
まずは、コードを上から読んだ時に「何を用意して、何を処理して、どこに反映するのか」を追えるようにする。

['仕事', '勉強'] [] は配列 複数の値を順番に入れる。
{ id: 1, title: 'Todo' } {} はオブジェクト 名前付きの情報をまとめる。
createTodo('買い物') () は渡す値 関数に値を渡す場所。
() => { ... } => は関数 短く関数を書く形。
1. 値を用意する 文字、数字、true/false、配列、オブジェクトを変数に入れる。
2. 処理をまとめる 関数にして、同じ処理を何度も使えるようにする。
3. 画面を動かす ボタン、入力欄、表示エリアをJavaScriptから操作する。
4. APIとつなぐ fetch、async、awaitでLaravel APIなどと通信する。

このサイト内での役割: JavaScriptはReactの画面、API通信、ボタン操作を書くための土台。 記号が分からない時は コードの記号、 どこにつながるかは 全体の流れ で確認できる。
