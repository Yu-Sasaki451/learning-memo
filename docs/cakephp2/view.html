Viewの書き方

Viewは、HTML、表示変数、フォーム、リンク、ページングを見る場所です。
CakePHP 2.0のViewファイルは .ctp で、PHPをそのまま書く形がよく出ます。

CakePHP2特有の書き方

CakePHP2のViewは、Bladeではなく .ctp にPHPを混ぜて書きます。
Laravelの {{ $user->name }} とは見た目が違うので、先に読み替えます。

<?php ... ?> HTMLの中でPHPを実行する範囲。Bladeの @foreach や {{ }} と違い、PHPを直接書く。
echo 画面に出力する命令。<?php echo h($name); ?> は「$nameを画面に出す」。
h() HTML用にエスケープする関数。Laravelの {{ }} が自動でしてくれる処理に近い。
$this->Form->input('name') FormHelperでname入力欄を作る。送信後はControllerの $this->request->data['User']['name'] 側へつながる。
array('label' => '名前') 入力欄の設定。ここでは「label表示は名前にする」という意味。

Viewコードの読み替え

Viewでは「Controllerから渡された変数を、HTMLのどこへ出しているか」を読みます。
画面に見えている文字、フォーム項目、リンク先はctpに書かれていることが多いです。

<?php echo ... ?> PHPの値をHTMLへ出す。「この値を画面に表示する」という意味。
h(...) 表示前に危険な文字を逃がす。ユーザー入力をそのままHTMLにしないため。
$user['User']['name'] ControllerやModelから来た配列。Userモデルのnameカラムを見る。
$this->Form CakePHPのフォーム作成用Helper。入力欄、送信ボタン、エラー表示を作る。

Controllerから来た変数を表示する

Controllerで $this->set('user', $user) した値は、Viewで $user として使います。
画面に出す時は、基本的に h() でエスケープします。

日本語に直すと

「Controllerから渡されたuserの名前を見出しに出し、メールアドレスを段落に出す」です。

参照元はControllerの $this->set('user', $user) です。setしていない名前はViewで使えません。

<h2><?php echo h($user['User']['name']); ?></h2>
<p><?php echo h($user['User']['email']); ?></p>
$user Controllerから渡された変数。
['User']['name'] CakePHP2の配列形式。Model名の下にカラム名が入る。
h() HTMLとして危険な文字を無害化して表示する。
echo 画面へ出力する。

foreachで一覧表示

一覧画面では、Controllerから渡された配列を foreach で回します。
CakePHP2では、1件ごとの中にもModel名のキーが入ることを意識します。

日本語に直すと

「usersの中身を1件ずつuserとして取り出し、ID、名前、メールをテーブルの1行として表示する」です。

参照元はControllerの $this->set('users', $users) です。Modelの find('all') やPaginatorで取った複数件が入ります。

<table>
    <thead>
        <tr>
            <th>ID</th>
            <th>名前</th>
            <th>メール</th>
        </tr>
    </thead>
    <tbody>
        <?php foreach ($users as $user): ?>
            <tr>
                <td><?php echo h($user['User']['id']); ?></td>
                <td><?php echo h($user['User']['name']); ?></td>
                <td><?php echo h($user['User']['email']); ?></td>
            </tr>
        <?php endforeach; ?>
    </tbody>
</table>
foreach 複数件の配列を1件ずつ処理する。
endforeach ViewでHTMLと混ぜやすいPHPの書き方。

ifで表示を切り替える

権限、状態、データ有無によって表示を変える時に使います。
画面にボタンが出ない時は、Viewのif条件とControllerから渡された値を確認します。

日本語に直すと

「Userのstatusがactiveなら有効と表示する。そうでなければ停止中と表示する」です。

ボタンや文言が出ない時は、この条件に使っている $user['User']['status'] の中身をControllerやDBで確認します。

<?php if ($user['User']['status'] === 'active'): ?>
    <span class="status status-active">有効</span>
<?php else: ?>
    <span class="status status-stopped">停止中</span>
<?php endif; ?>
=== 値と型が同じか確認する比較。文字列のactiveと一致するか見ている。
else 条件に当てはまらない時の表示。
endif ViewでHTMLと混ぜやすいPHPのif終了。

FormHelperでフォームを書く

CakePHP2では、フォームを $this->Form で書くことが多いです。
Controllerの $this->request->data とModelのバリデーションにつながります。

日本語に直すと

「User用のフォームを開始し、nameとemailの入力欄を作り、保存ボタンでフォームを閉じる」です。

このフォームを送信すると、Controllerの $this->request->data に User.name と User.email が入ります。

<?php echo $this->Form->create('User'); ?>
<?php echo $this->Form->input('name', array('label' => '名前')); ?>
<?php echo $this->Form->input('email', array('label' => 'メールアドレス')); ?>
<?php echo $this->Form->end('保存'); ?>
Form->create('User') Userモデル用のフォームを開始する。入力値を User というキーの下にまとめるための指定。
Form->input('name') name項目の入力欄を作る。参照先はModelの User.name と、バリデーションの $validate['name']。
label 画面に表示する項目名。
Form->end('保存') 送信ボタンとform終了タグを作る。ここで押された内容がControllerのaddやeditへPOSTされる。

リンクと削除ボタン

画面遷移は HtmlHelper、削除のようにPOSTで送りたいものは FormHelper を使うことが多いです。
削除を普通のaタグだけで作っていないか確認します。

日本語に直すと

「編集リンクを作る。クリックしたらusers controllerのedit actionへ、今のUserのidを付けて移動する。削除は確認メッセージを出してからPOSTでdelete actionへ送る」です。

参照先はControllerの edit($id) と delete($id) です。

<?php
echo $this->Html->link(
    '編集',
    array('controller' => 'users', 'action' => 'edit', $user['User']['id'])
);
?>

<?php
echo $this->Form->postLink(
    '削除',
    array('action' => 'delete', $user['User']['id']),
    array('confirm' => '本当に削除しますか？')
);
?>
Html->link 通常のリンクを作る。
postLink POST送信するリンク風の操作。削除でよく使う。
confirm 実行前の確認メッセージ。

フラッシュメッセージ

Controllerで Session->setFlash() したメッセージは、ViewやLayout側で表示します。
保存成功メッセージが出ない時は、Layoutに表示処理があるか見ます。

日本語に直すと

「Controllerがセットした一時メッセージを、この場所に表示する」です。

参照元はControllerの $this->Session->setFlash('...') です。Viewにflashがなければ、メッセージをセットしても画面に出ません。

<?php echo $this->Session->flash(); ?>
Session->flash() 一時メッセージを画面に表示する。
探す場所 app/View/Layouts/default.ctp に書かれていることが多い。

ページング

一覧でページ移動や並び替えがある場合は、ControllerのPaginator設定とViewのPaginatorHelperをセットで見ます。

日本語に直すと

「名前の見出しをクリックで並び替えできるようにする。下に前へ、ページ番号、次へのリンクを出す」です。

参照元はControllerの $this->Paginator->paginate('User') と $paginate 設定です。

<th><?php echo $this->Paginator->sort('User.name', '名前'); ?></th>

<div class="paging">
    <?php echo $this->Paginator->prev('前へ'); ?>
    <?php echo $this->Paginator->numbers(); ?>
    <?php echo $this->Paginator->next('次へ'); ?>
</div>
sort クリックで並び替えできる見出し。
prev / next 前ページ、次ページのリンク。
numbers ページ番号のリンク。

ElementとLayout

ヘッダー、サイドバー、共通パーツは、今見ているctpに直接ないことがあります。
その場合はLayoutやElementを探します。

日本語に直すと

「user_menuという共通部品を読み込み、その部品へuser変数を渡す」です。

参照先は app/View/Elements/user_menu.ctp です。今見ているViewにHTMLがなくても、Element側に書かれていることがあります。

<?php echo $this->element('user_menu', array('user' => $user)); ?>
element('user_menu') app/View/Elements/user_menu.ctp を読み込む。
Layouts/default.ctp ページ全体の共通HTML。ヘッダーやフッターがあることが多い。

Laravel Bladeなら近い形

<h2>{{ $user->name }}</h2>

@foreach ($users as $user)
    <p>{{ $user->email }}</p>
@endforeach

<form method="POST" action="{{ route('users.store') }}">
    @csrf
    <input type="text" name="name" value="{{ old('name') }}">
    <button type="submit">保存</button>
</form>
CakePHP 2.0 <?php echo h(...) ?>、FormHelper、HtmlHelper をよく見る。
Laravel {{ ... }}、@foreach、route()、@csrf をよく見る。
