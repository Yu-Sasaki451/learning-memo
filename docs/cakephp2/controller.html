Controllerの書き方

Controllerは、URLから呼ばれる処理の入口です。
画面表示、入力受け取り、Model呼び出し、保存後のリダイレクトを読む場所です。

先に覚える読み替え

Controllerのコードは、英語の命令文をCakePHPの形で書いているだけです。
読む時は、記号を先に日本語へ直すと追いやすくなります。

$this 「今このController自身」という意味。UsersController内なら、UsersControllerから見た部品やModelを呼ぶ。
-> 「左のものが持っている右の機能を使う」という意味。$this->User は「このControllerが持つUserモデル」。
array('key' => 'value') 設定のまとまり。key が項目名、value が中身。Laravelの連想配列と同じ感覚で読む。
if (...) 「もし条件に当てはまるなら」。保存できた時、POSTされた時、データが空の時などを分岐する。
! 否定。「そうではないなら」。!$this->User->exists($id) は「そのidのUserが存在しないなら」。
return そこで処理を返して終わる。redirectの前にreturnがある時は、移動して以降の処理を続けない。

Controllerの基本形

CakePHP 2.0では、Controller名は複数形にすることが多いです。
UsersController なら、基本的に User モデルと app/View/Users を使います。

日本語に直すと

「UsersControllerという画面処理の置き場を作る。Session、Paginator、Html、Formを使えるようにする。indexが呼ばれたらUserテーブルから全件取り、Viewへusersという名前で渡す」です。

参照している先は、app/Controller/AppController.php、app/Model/User.php、app/View/Users/index.ctp です。

App::uses('AppController', 'Controller');

class UsersController extends AppController {
    public $components = array('Session', 'Paginator');
    public $helpers = array('Html', 'Form', 'Paginator');

    public function index() {
        $users = $this->User->find('all');
        $this->set('users', $users);
    }
}
App::uses 使うクラスを読み込む。古いCakePHP2のコードでよく見る。
extends AppController 共通Controllerを継承する。ログイン判定などはAppController側にあることが多い。
$components Controller側で使う部品を登録する。Session はメッセージやログイン情報、Paginator はページング処理で使う。
$helpers View側で使う部品を登録する。ここにある名前が $this->Form や $this->Html としてctpで使われる。
index() URLのactionに対応するメソッド。/users/index や /users から呼ばれ、終わると app/View/Users/index.ctp が表示される。

一覧表示 index

一覧画面では、ページング付きで取得する形がよく出ます。
$this->paginate に条件を書き、$this->Paginator->paginate() で取得します。

日本語に直すと

「Userを20件ずつ、作成日の新しい順で、削除されていないものだけ取る。取った結果をusersという名前でViewへ渡す」です。

この形にする理由は、Controller内に一覧取得の条件をまとめ、View側では $users を表示するだけにするためです。

public $paginate = array(
    'User' => array(
        'limit' => 20,
        'order' => array('User.created' => 'desc'),
        'conditions' => array('User.deleted' => 0)
    )
);

public function index() {
    $users = $this->Paginator->paginate('User');
    $this->set('users', $users);
}
limit 1ページに何件表示するか。
order 並び順。createdの新しい順など。
conditions 取得条件。削除済みを除外するなど。
set('users', $users) Controller内の $users を、View内でも $users という名前で使えるようにする。

詳細表示 view

詳細画面では、idを受け取って1件だけ取得します。
見つからない時は、何も表示せずに404へ流す形がよく使われます。

日本語に直すと

「URLから来たidでUserを1件探す。見つからなければ404にする。見つかったらuserという名前でViewへ渡す」です。

conditions はSQLのWHERE条件に近いです。ここでは「User.idがURLのidと同じもの」という条件です。

public function view($id = null) {
    $user = $this->User->find('first', array(
        'conditions' => array('User.id' => $id)
    ));

    if (empty($user)) {
        throw new NotFoundException('ユーザーが見つかりません');
    }

    $this->set('user', $user);
}
$id = null URLからidが来ない場合も考えて、初期値をnullにしている。/users/view/5 なら5が入る。
find('first') 1件だけ取得する。
throw new NotFoundException 該当データがない時に404ページへする。

新規登録 add

登録処理では、POSTで送られた時だけ保存します。
保存成功なら一覧や詳細へ移動し、失敗なら同じ画面に戻して入力エラーを見せます。

日本語に直すと

「登録フォームが送信された時だけ、Userを新規登録モードにして、フォームの入力値を保存する。保存できたらメッセージを出して一覧へ移動する。保存できなければ入力確認メッセージを出す」です。

$this->request->data は、ViewのFormHelperから送られた入力値です。たとえば User.name と User.email が入ります。

public function add() {
    if ($this->request->is('post')) {
        $this->User->create();

        if ($this->User->save($this->request->data)) {
            $this->Session->setFlash('ユーザーを登録しました');
            return $this->redirect(array('action' => 'index'));
        }

        $this->Session->setFlash('入力内容を確認してください');
    }
}
$this->request->is('post') フォーム送信された時だけ保存するための判定。最初に登録画面を開いただけの時は保存しない。
create() 新規登録として保存する準備。
$this->request->data フォームから送られた入力値。参照元は app/View/Users/add.ctp の Form->input()。
save() Modelのバリデーションを通してDBへ保存する。
return $this->redirect() 保存後に別画面へ移動する。returnを書くと、その後の処理に進まない。

編集 edit

編集では、最初に既存データを表示し、送信されたら保存します。
CakePHP2の古いコードでは $this->request->data に既存データを入れて、FormHelperに初期値を出させる形がよく出ます。

日本語に直すと

「まずidのUserが存在するか確認する。フォームが送信されたら入力値を保存する。まだ送信されていない初回表示なら、DBから既存データを取ってフォームに入れる」です。

else 側は「保存失敗時」ではありません。初めて編集画面を開いた時に、既存の名前やメールをフォームに表示するための処理です。

public function edit($id = null) {
    if (!$this->User->exists($id)) {
        throw new NotFoundException('ユーザーが見つかりません');
    }

    if ($this->request->is(array('post', 'put'))) {
        if ($this->User->save($this->request->data)) {
            $this->Session->setFlash('ユーザーを更新しました');
            return $this->redirect(array('action' => 'index'));
        }

        $this->Session->setFlash('入力内容を確認してください');
    } else {
        $this->request->data = $this->User->find('first', array(
            'conditions' => array('User.id' => $id)
        ));
    }
}
exists($id) そのidのデータが存在するか確認する。
array('post', 'put') 登録系のPOSTと、更新系のPUTを受ける。CakePHPのFormHelperが編集フォームでPUT相当を送ることがあるため、この形で見る。
else 送信前の初回表示。DBから既存データを取り、フォームへ入れる。
$this->request->data = ... FormHelperが初期値として使うデータをセットする。Viewの Form->input('name') がこの値を見て入力欄へ表示する。

削除 delete

削除はGETで実行させないのが基本です。
リンクを開いただけで削除される事故を避けるため、POSTやDELETEだけ許可します。

日本語に直すと

「この削除処理はPOSTかDELETEで来た時だけ許可する。Userを削除できたら成功メッセージ、できなければ失敗メッセージを出す。最後に一覧へ戻る」です。

参照元はViewの Form->postLink() や削除フォームです。普通のリンクで削除しないために、この形になっています。

public function delete($id = null) {
    $this->request->allowMethod('post', 'delete');

    if ($this->User->delete($id)) {
        $this->Session->setFlash('ユーザーを削除しました');
    } else {
        $this->Session->setFlash('削除できませんでした');
    }

    return $this->redirect(array('action' => 'index'));
}
allowMethod 許可するHTTPメソッドを制限する。
delete($id) 指定したidのデータを削除する。

ログイン許可や別Modelの利用

共通処理は beforeFilter() にあることが多いです。
Controller名と違うModelを使う場合は、loadModel() を探します。

日本語に直すと

「このControllerの各actionが動く前に、親の共通処理を実行する。loginとlogoutはログイン前でも許可する。dashboardではOrderモデルを追加で読み込み、ログイン中ユーザーの注文を取得してViewへ渡す」です。

parent::beforeFilter() は特に重要です。AppController側のログイン設定や共通設定を飛ばさないために呼びます。

public function beforeFilter() {
    parent::beforeFilter();
    $this->Auth->allow('login', 'logout');
}

public function dashboard() {
    $this->loadModel('Order');

    $orders = $this->Order->find('all', array(
        'conditions' => array('Order.user_id' => $this->Auth->user('id'))
    ));

    $this->set('orders', $orders);
}
parent::beforeFilter() AppController側の共通処理も実行する。消すとログイン判定などが壊れることがある。
Auth->allow ログイン前でも見られるactionを指定する。
loadModel('Order') このControllerの標準Model以外を使う。
Auth->user('id') ログイン中ユーザーのidを取る。

Laravelなら近い形

日本語に直すと

CakePHP2の $this->User->save($this->request->data) は、Laravelでいう「Requestで受け取った入力値をUserへ保存する」に近いです。

ただし、CakePHP2では入力値が $this->request->data に入り、バリデーションはModelの $validate にあることが多い点が違います。

public function store(Request $request)
{
    $user = User::create($request->validated());

    return redirect()->route('users.index')
        ->with('status', 'ユーザーを登録しました');
}
CakePHP 2.0 $this->request->data、$this->User->save()、$this->Session->setFlash() をよく見る。
Laravel Request、User::create()、with() をよく見る。
